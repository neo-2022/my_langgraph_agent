## 2) Текущая задача (ONE focus)
**Задача:** Сделать Debugger MVP, который позволяет быстро диагностировать проблемы UI/Graph/Run без “гадания”.

**Определение готовности (DoD) — Debugger MVP:**
- [ ] В UI есть доступная панель/вкладка “Debug” (или drawer), которая не мешает основному UX.
- [ ] Debug показывает “снимок состояния” (snapshot) одним блоком, пригодным для копирования:
  - [ ] текущая вкладка/режим (run/graph/split), размеры split (если применимо)
  - [ ] assistantId/run_id (если есть)
  - [ ] состояние загрузки/последней ошибки GraphView
  - [ ] количество nodes/edges, timestamp последнего успешного построения
  - [ ] параметры визуализации (direction и т.п.)
- [ ] Есть кнопки:
  - [ ] “Refresh graph” (явно повторить построение/запрос графа)
  - [ ] “Copy debug snapshot”
  - [ ] (опционально) “Toggle direction” если это сейчас используется как workaround
- [ ] Любая ошибка запроса/парсинга/рендера графа отображается в Debug с понятным текстом + коротким контекстом (endpoint/статус/время).

**Как воспроизвести проблему (первый кейс для дебаггера): “пустой граф”**
1) Открыть UI → перейти в Graph (Graph-only) или включить Split.
2) Иногда граф пустой (nodes/edges = 0 или рендер не происходит).
3) Ноды появляются только если: обновить страницу / нажать “обновить граф” / сменить направление.

**Ожидаемое поведение:** граф всегда строится автоматически, без ручных “пинков”.
**Фактическое поведение:** иногда требуется ручной refresh/смена direction.
