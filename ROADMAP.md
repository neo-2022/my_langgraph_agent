## 0) Цель проекта (куда идём)

Сделать **локального агента** на LangGraph, который:
- работает с **Ollama** (локальные модели)
- умеет подключать **облачные модели** (OpenAI/Anthropic и др.) **в отдельном окружении** (без конфликтов)
- поддерживает **инструменты** (файлы, shell, web-поиск)
- имеет **локальный интерфейс**, не хуже Studio по удобству:
  - граф (узлы/стрелки)
  - запуски (runs), история, потоки событий
  - пошаговое выполнение (model → tools → model)
  - state/config/context, diff state
  - таймлайны (время узлов/инструментов), метрики токенов

В перспективе:
- интеграция с **Open Interpreter** (узел, который генерирует/исполняет код)
- агент сможет **сам создавать новые инструменты по запросу** (OpenClaw-style)

## 3) План работ (по этапам)

## Этап A — React UI (минимально полезный “мини-Studio”)
Цель: начать работать с графом удобно, без внешнего Studio.

Сделать:
1) Папка `ui/` (Vite + React)
2) Левый **выдвижной** сайдбар (панель настроек):
   - выбор модели
   - tool_calls таблица
   - restart langgraph
3) Основная зона:
   - вкладка “Run” (чат):
     - ввод текста
     - запуск через `/runs/stream` (стрим событий)
     - вывод ответа
     - отображение tool_calls + tool results
4) Бэкенд-прокси в FastAPI:
   - эндпоинты вида `/api/...` → дергают `127.0.0.1:2024`
   - единая точка входа для UI

Критерий готовности:
- можно запустить запрос из UI
- видеть шаги model/tools (хотя бы списком событий)

## Этап I — Open Interpreter (узел генерации и исполнения кода)
Цель: агент может генерировать/запускать код для задач.

Сделать:

  - принимает задачу
  - генерирует код
  - выполняет локально (с ограничениями)
- безопасность:
  - sandbox, таймауты, ограничение директорий
- UI: показывать код, вывод, ошибки, подтверждения (human-in-the-loop)

---

## Этап J — Self-Tooling / агент создаёт новые инструменты сам
Цель: “агент пишет инструмент под задачу”.

Сделать:
- pipeline:
  1) агент предлагает новый tool (описание + сигнатура)
  2) генерирует код инструмента
  3) запускает тест (минимальный)
  4) регистрирует tool в проекте (tools registry)
  5) повторно запускает задачу уже с новым tool
- UI: мастер создания инструмента + кнопки “принять/отклонить”

## 4) Принципы разработки (важно)

- “Одна команда — один шаг” (для новичка)
- Никаких секретов в репозитории:
  - `.env` не коммитим
  - токены не вставляем в чат/файлы
- Никаких конфликтов зависимостей:
  - облако — отдельное окружение
- UI должен быть локальным и удобным:
  - граф + трассировка + state
